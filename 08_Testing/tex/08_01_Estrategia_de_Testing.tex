\section{Estrategia de testing}

\subsection{Descripcion}

Este documento reservara la estrategia general de pruebas del proyecto.

\subsection{Alcance}

Incluira pruebas unitarias, integracion, end to end y validacion operativa.

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{Estrategia de testing}: forma planeada de validar calidad por nivel y riesgo.
\end{itemize}

\subsection{Reglas}

\begin{itemize}
  \item El testing debe priorizar reglas del dominio y flujos criticos.
  \item Los escenarios que puedan generar disputa deben tener cobertura prioritaria.
\end{itemize}

\subsection{Casos borde}

\begin{itemize}
  \item Pendiente de desarrollar por capa y herramienta.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item No se considera listo un flujo critico sin criterio de prueba.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar distribucion final de cobertura por nivel de prueba.
\end{itemize}
