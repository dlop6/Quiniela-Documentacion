\section{Estado, validacion y datos}

\subsection{Descripcion}

Este documento reservara la estrategia de estado remoto, validacion y consumo de datos del frontend.

\subsection{Alcance}

Aplica a contratos cliente-servidor y manejo de estado observable.

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{Estado remoto}: informacion obtenida desde backend y sincronizada en el frontend.
\end{itemize}

\subsection{Reglas}

\begin{itemize}
  \item Los contratos del frontend deben validarse con las mismas reglas funcionales del sistema.
  \item El frontend debe diferenciar datos remotos, estado local de UI y validacion de inputs.
\end{itemize}

\subsection{Casos borde}

\begin{itemize}
  \item Pendiente de aterrizar junto con los endpoints conceptuales del backend.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item El frontend no asume datos no garantizados por el backend.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar convenciones finales de queries, mutaciones y esquemas compartidos.
\end{itemize}
