# Example

Task: assess whether a high-impact change touching backend, integrations, and admin behavior is operationally ready.

Expected workflow:

1. Read operational, testing, and planning docs first.
2. Read the changed-area docs.
3. Identify unresolved decisions, coverage gaps, and traceability risks.
4. Evaluate fallback and continuity sensitivity.
5. Report release or continuity risk together with blocking gaps.
