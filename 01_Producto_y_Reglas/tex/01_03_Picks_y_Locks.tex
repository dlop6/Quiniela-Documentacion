\section{Picks y locks}

\subsection{Descripcion}

Este documento define las reglas funcionales de captura, validacion, bloqueo y visibilidad de los picks del sistema. Su objetivo es dejar sin ambiguedad que puede ingresarse, hasta cuando puede modificarse y bajo que condiciones deja de ser visible o editable.

\subsection{Alcance}

Aplica a:

\begin{itemize}
  \item picks por partido;
  \item predicciones especiales de Campeon y Goleador;
  \item reglas de lock asociadas a ambos tipos de prediccion.
\end{itemize}

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{Ventana abierta}: periodo durante el cual una prediccion puede crearse o editarse.
  \item \textbf{Lock}: momento a partir del cual una prediccion deja de ser editable.
  \item \textbf{Picks por partido}: predicciones repetibles asociadas a partidos individuales.
  \item \textbf{Predicciones especiales}: predicciones unicas de torneo para Campeon y Goleador.
  \item \textbf{Sin pick}: ausencia valida de prediccion guardada antes del lock.
\end{itemize}

\subsection{Reglas}

\subsubsection{Reglas generales}

\begin{enumerate}
  \item Solo usuarios en estado \texttt{ACTIVO} pueden guardar picks validos.
  \item Toda validacion de lock debe depender del kickoff oficial vigente en el sistema y no del reloj local del navegador.
  \item El backend debe rechazar cualquier modificacion posterior al lock, aunque la interfaz intente enviarla.
\end{enumerate}

\subsubsection{Picks por partido}

\begin{enumerate}
  \item Cada pick por partido se compone de dos valores enteros: goles local y goles visita.
  \item El rango valido para ambos valores es de 0 a 9 inclusive.
  \item Mientras el partido este en ventana abierta, el participante puede crear o editar su pick.
  \item El lock del pick por partido ocurre automaticamente 5 minutos antes del kickoff oficial vigente del partido.
  \item Si no existe pick guardado al llegar el lock, el partido queda funcionalmente sin pick y otorga 0 puntos.
\end{enumerate}

\subsubsection{Predicciones especiales}

\begin{enumerate}
  \item Las predicciones especiales forman parte del modulo funcional de picks.
  \item Las predicciones especiales se capturan en un flujo separado del flujo de picks por partido.
  \item Las predicciones especiales pueden crearse o editarse hasta 5 minutos antes del kickoff oficial del primer partido del torneo.
  \item A partir de ese momento, las predicciones especiales quedan bloqueadas.
\end{enumerate}

\subsubsection{Visibilidad}

\begin{enumerate}
  \item Los picks de otros usuarios permanecen ocultos mientras el partido no haya cerrado.
  \item Los picks de otros usuarios solo se vuelven visibles despues de que el partido ya haya cerrado.
  \item Este documento no fija aun la politica final de visualizacion de predicciones especiales frente a otros usuarios; esa decision se cerrara cuando se refine la visualizacion del leaderboard y perfiles.
\end{enumerate}

\subsection{Casos borde}

\begin{itemize}
  \item Si el kickoff cambia antes del lock, el cierre debe recalcularse con la nueva hora oficial.
  \item Si el kickoff cambia despues del lock por una postergacion, el pick ya bloqueado permanece valido salvo decision administrativa extraordinaria auditada.
  \item Si el usuario guarda parcialmente una jornada, solo cuentan los partidos cuya prediccion haya sido persistida antes del lock correspondiente.
  \item Si el sistema no encuentra un pick guardado para un partido al llegar el lock, no debe generar una prediccion implicita ni un valor por defecto.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item No existe edicion valida de picks por partido despues del lock.
  \item No existe edicion valida de predicciones especiales despues de su fecha de cierre.
  \item La visibilidad de picks ajenos nunca ocurre antes del cierre del partido correspondiente.
  \item Un partido sin pick no genera prediccion automatica.
  \item Ningun valor de goles fuera del rango 0 a 9 es valido.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar la politica exacta de visibilidad de Campeon y Goleador frente a otros usuarios.
\end{itemize}

\begin{itemize}

  \item `Decision pendiente` `No bloqueante`: confirmar si el estado funcional `sin pick` se documentara tambien como estado persistido explicito o solo como resultado de negocio.
\end{itemize}
