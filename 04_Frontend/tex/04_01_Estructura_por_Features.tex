\section{Estructura por features}

\subsection{Descripcion}

Este documento reservara la estructura funcional del frontend basada en features y capas claras.

\subsection{Alcance}

Aplica al repositorio frontend y a sus criterios de organizacion.

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{Feature}: agrupacion funcional del frontend con UI, estado y acceso a datos relacionados.
\end{itemize}

\subsection{Reglas}

\begin{itemize}
  \item La estructura del frontend se describira por features.
  \item `shared` solo contendra piezas realmente reutilizables.
  \item La organizacion no debe ocultar reglas de negocio criticas en componentes UI.
\end{itemize}

\subsection{Casos borde}

\begin{itemize}
  \item Pendiente de cerrar cuando exista un inventario real de features.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item La estructura del frontend no redefine contratos del dominio.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar mapa final de features para v1.
\end{itemize}
