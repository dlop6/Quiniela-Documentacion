\section{Procesos de negocio}

\subsection{Descripcion}

Este documento reservara los procesos backend clave: activacion, lock, scoring, sincronizacion, recalculo y archivado.

\subsection{Alcance}

Describe secuencias funcionales del backend, no implementacion de codigo.

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{Proceso}: secuencia de pasos del sistema para cumplir una regla funcional.
\end{itemize}

\subsection{Reglas}

\begin{itemize}
  \item Cada proceso debe ser trazable y consistente con el dominio.
  \item Los procesos criticos deben describir triggers, validaciones y resultado esperado.
\end{itemize}

\subsection{Casos borde}

\begin{itemize}
  \item Pendiente de desarrollar por proceso en iteraciones posteriores.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item Un proceso critico no puede quedar sin criterio de error ni de auditoria.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar priorizacion interna de procesos para documentacion detallada.
\end{itemize}
