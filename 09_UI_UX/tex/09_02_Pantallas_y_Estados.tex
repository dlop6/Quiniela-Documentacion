\section{Pantallas y estados}

\subsection{Descripcion}

Este documento reservara el inventario de pantallas y sus estados visibles.

\subsection{Alcance}

Aplica a vistas publicas, privadas y administrativas.

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{Estado de pantalla}: combinacion de datos y permisos que altera el comportamiento visible de una vista.
\end{itemize}

\subsection{Reglas}

\begin{itemize}
  \item Cada pantalla debe declarar estados vacios, cargando, error y exito cuando aplique.
  \item Los estados visibles deben alinearse con estados reales del backend.
\end{itemize}

\subsection{Casos borde}

\begin{itemize}
  \item Pendiente de desarrollar junto con flujos frontend y contratos conceptuales.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item El frontend no debe mostrar un estado funcional que el backend no pueda sostener.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar inventario definitivo de pantallas por rol.
\end{itemize}
