\section{Flujos y paginas}

\subsection{Descripcion}

Este documento reservara la definicion de rutas, paginas y flujos principales de usuario y admin.

\subsection{Alcance}

Describe navegacion y comportamiento funcional, no diseno visual detallado.

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{Flujo}: recorrido funcional completo de un usuario para lograr un objetivo.
\end{itemize}

\subsection{Reglas}

\begin{itemize}
  \item Cada flujo debe alinearse con las reglas cerradas del dominio.
  \item Las paginas deben declarar claramente su accion principal.
\end{itemize}

\subsection{Casos borde}

\begin{itemize}
  \item Pendiente de aterrizar cuando se cierre el detalle de estados de pantalla.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item Un flujo frontend no puede permitir acciones que backend deba prohibir, como editar picks post-lock.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar inventario final de paginas publicas, privadas y administrativas.
\end{itemize}
