\section{Criterios de aceptacion}

\subsection{Descripcion}

Este documento reservara los criterios de aceptacion funcional y UAT del proyecto.

\subsection{Alcance}

Aplica a validacion final con usuarios y responsables del proyecto.

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{UAT}: validacion con usuarios o responsables antes de considerar un flujo aceptado.
\end{itemize}

\subsection{Reglas}

\begin{itemize}
  \item Un criterio de aceptacion debe ser observable y verificable.
  \item Los criterios deben alinearse con reglas del dominio y objetivos operativos.
\end{itemize}

\subsection{Casos borde}

\begin{itemize}
  \item Pendiente de aterrizar por flujo y prioridad.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item No se aceptan criterios ambiguos ni dependientes de interpretacion libre.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar matriz final de aprobacion por flujo.
\end{itemize}
