\section{Casos criticos de prueba}

\subsection{Descripcion}

Este documento reservara el inventario de escenarios de negocio que no deben romperse.

\subsection{Alcance}

Aplica a activacion, picks, lock, resultados, scoring, leaderboard y admin.

\subsection{Definiciones}

\begin{itemize}
  \item \textbf{Caso critico}: escenario cuya falla afecta justicia, operacion o confianza del sistema.
\end{itemize}

\subsection{Reglas}

\begin{itemize}
  \item Los casos criticos se definiran por riesgo funcional, no solo por complejidad tecnica.
  \item Los escenarios de lock y scoring tendran prioridad alta.
\end{itemize}

\subsection{Casos borde}

\begin{itemize}
  \item Pendiente de aterrizar con base en el dominio ya cerrado.
\end{itemize}

\subsection{Invariantes}

\begin{itemize}
  \item Ninguna regla critica del juego debe quedar sin un caso de prueba asociado.
\end{itemize}

\subsection{Preguntas abiertas}

\begin{itemize}
  \item Confirmar la lista final de escenarios minimos para salida a operacion.
\end{itemize}
